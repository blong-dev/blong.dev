Platform

The foundation everything else on this page runs on.

Gnosis deployed
The private, event-driven AI platform behind everything else here.

Public and private data flows through an ingestion layer into a knowledge graph and a time-series warehouse, where an autonomous agent layer reasons over it and ships results to a live dashboard. Go services move work across a message bus where every agent action is an auditable event — no agent calls another directly. Postgres, Neo4j, Elasticsearch, and Redis sit behind it, held together by ~2,200 automated tests. It runs on two consumer desktops over a 10 GbE link — most of the intelligence is local, so there's no per-token bill for the volume work. I set direction; the platform runs the loop.

The workforce

A named team, not a pile of scripts — one custom, bus-native harness running persistent, ephemeral, and non-LLM agents. Every action is a logged, replayable event.

Cosmo
The agent that runs the other agents.
Chief of staff · frontier (Claude)

A persistent agent that manages my attention the way a chief of staff manages an executive's: it triages what reaches me, delegates to specialist agents, and keeps a running memory of decisions and procedures so context survives across days. ~38 tools (filesystem, the knowledge graph, memory, task + kanban management); runs on a frontier model because this is the judgment seat. Everything it does is a logged bus event.

Orca
Triage for a firehose of signals.
Orchestrator · local 27B

Sits between the raw event bus and Cosmo, deciding in real time what's noise, what's an escalation, and what should become work — then dispatching it. Runs on the local 27B model (cheap, always-on), with deterministic durability sweeps so nothing stalls silently. Designed to fail safe — rate-capped, deduplicated, with a degraded-mode brake — because an orchestrator that can dispatch is an orchestrator that can flood.

Eagle
Business profiling and outreach, as an autonomous agent.
Sales-intel engine · local 27B / SDK

Builds business profiles, manages an outreach pipeline, runs briefings. It exists in two forms — local 27B and a frontier backend via the Agent SDK — the same agent, swapped by a config line. Structured by contract: the agent returns typed JSON and code does the writes, so nothing depends on the model calling a tool correctly.

Scout
An internal agent capability, shipped as its own product.
Sales-intel SaaS · CF Workers + D1

A standalone SaaS surface (Hono + React on Cloudflare Workers, D1) that turns the platform's business-profiling into a usable app — campaigns, leads, outreach — on its own domain. A clean example of productizing an internal capability without rebuilding it.

Sentinel
The agent that watches the other agents.
SRE · TypeScript / Node

A monitoring service that watches the platform like an SRE: Docker events, consumer lag on the bus, model health. Its design principle is hard-won — measure liveness by real work progress, not heartbeats — after green health checks once masked a stalled pipeline. It's why a 20-agent system run by one person doesn't quietly fall over.

Apps

Shipped on the platform. Live ones link out.
Tail Atlas deployed
One clean picture of the world fleet, from sources that all disagree.

710,000 aircraft and 313,000 owners served at the edge (1.3M+ in the full pipeline), assembled from ~25 aviation registries plus live position data. A Python pipeline normalizes them over the bus into Postgres; a React + Hono app serves it from Cloudflare Workers, D1, and R2 — fast and cheap, with 435 tests. The volume isn't the hard part — making tail numbers, airframes, and type certificates agree across contradictory sources with no hand-curation is. That resolution layer is the product.
Telekora deployed
Generate a whole course — slides, images, quizzes — from a prompt.

A multi-tenant B2B LMS where each tenant brings their own model: per-tenant LLM keys are envelope-encrypted in Cloudflare Secrets Store, and course generation uses forced tool-use against a strict schema, so the output is always valid JSON — never a parsing gamble. A Hono API on Cloudflare Workers with D1, R2, and KV; Stripe billing, Google auth, and a Telegram Mini App front end. A complete SaaS, not a demo.
HometownWire in flight
Turning the public record into something you can actually search.

Council meetings, votes, and speaker-labeled transcripts, made searchable and readable. A React + Hono app on Cloudflare Workers over a civic corpus in D1 — and the data lands through the same platform pipeline as everything else, as additive handlers, so there's no separate stack to maintain. The hard part of civic data is sourcing and structuring it; the platform already does both.

Bamboo deployed
I didn't pick an agent framework. I built one.

The runtime under every agent above. It started as a containerized bot and got rebuilt until every agent action flows through the message bus and is auditable — no agent calls another directly, so the whole system is logged and replayable. One config line swaps an agent between local inference and a frontier model; a new agent is a manifest plus a prompt, no runtime code. It runs persistent, ephemeral, and multi-tenant agents on one substrate — the reason one person can operate a 20-agent organization.

otter-docs shipped · PyPI
Open-source codebase intelligence for any language.

A polyglot code-intelligence library — tree-sitter ASTs + cross-file symbol resolution + LLM-generated findings and diffs. MIT-licensed, on PyPI, 354 tests, and F1 0.854 on a Type-4 semantic-clone benchmark. Auto-docs and senior-dev-style review that run on any codebase, not just mine.

Six months, solo. Two machines, a bucket of domains, a Claude sub, and a dream.
